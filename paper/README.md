# Paper build folder — Learning-Enhanced SCUC

This folder is a self-contained project for rebuilding the revised manuscript.
All tables and figures are produced from data that already exists in
`../results_revision/` and `../results_prev2/figures/` — **no new solver runs
are required** to compile the paper.

## Layout

```
paper/
├── paper.tex                 # revised manuscript (LLNCS class)
├── response_to_reviewers.tex # response letter to R1/R2
├── data/                     # CSV sources (copied from results_revision/)
│   ├── speedup_summary.csv        # mean/median speedup + bootstrap 95% CI
│   ├── wilcoxon.csv               # paired Wilcoxon p-values for key modes
│   ├── constraint_reduction.csv   # initial vs retained post-contingency constraints
│   ├── large_case_summary.csv     # case1354pegase follow-up runtimes
│   ├── case57_audit.csv           # structural-infeasibility audit
│   └── headline.json              # short machine-readable summary
├── figures/                  # images included by paper.tex
│   ├── ecdf.png
│   ├── speedup_general.png
│   └── large_cases_runtime.png
├── tables/                   # LaTeX tables generated by scripts/postprocess.py
│   ├── speedup_with_ci.tex         # \input'd by paper.tex (Tab. 2')
│   ├── speedup_with_ci.csv         # same content as CSV for inspection
│   ├── gnn_vs_1nn.tex              # GNN vs 1-NN ablation (Tab. 5)
│   ├── gnn_vs_1nn.csv              # same content as CSV for inspection
│   ├── large_case.tex              # case1354pegase table (Tab. 6)
│   └── large_case_table.csv        # same content as CSV for inspection
└── scripts/
    └── postprocess.py        # re-generates tables/*.tex from data/*.csv
```

## How to rebuild

1. (Optional) Re-generate the three auto-built tables from the CSVs:

   ```bash
   cd paper
   python scripts/postprocess.py
   ```

   This writes `tables/speedup_with_ci.tex`, `tables/gnn_vs_1nn.tex`, and
   `tables/large_case.tex`. `paper.tex` `\input`s the first three files.

2. Compile the paper:

   ```bash
   cd paper
   pdflatex paper.tex
   pdflatex paper.tex   # second pass for references
   ```

3. Compile the response letter:

   ```bash
   cd paper
   pdflatex response_to_reviewers.tex
   ```

## What was changed relative to the previous revision

All changes are **additive** — no existing speedup number was modified.

1. **Abstract** — tightened the headline claim (``up to 25.8x on case89pegase
   and 8.6x on average'', plus a one-line mention of the case1354pegase
   follow-up). *Addresses R1-C2.*
2. **Introduction** — added a quantitative context paragraph comparing our
   best speedup to Xavier et al. (2019) and Jiménez-Cordero et al. (2022).
   *Addresses R1-C3.*
3. **Model Definition** — the base-case flow bounds (Eq. 12) and the
   post-contingency bounds (Eqs. 14–17) were rewritten in symmetric two-sided
   form with explicit `−F^N` / `−F^E` on the lower side. *Addresses R2-C1.*
4. **Acceleration Framework / GNN** — inserted Table 5 (GNN vs 1-NN
   ablation) and a two-sentence interpretation of why the GNN does not beat
   the 1-NN baseline. *Addresses R1-C5.*
5. **Acceleration Framework / BANDIT** — the candidate set
   `K = {64,128,256}`, the reward `R_t = −N^added_t`, and `ε = 0.1` are now
   stated. The paragraph now labels the bandit as a negative result.
   *Addresses R1-C6.*
6. **Acceleration Framework / LAZY** — added an explicit paragraph explaining
   why lazy enforcement is applied only to the post-contingency constraints
   and not to the base-case constraints, and noting that a preliminary
   experiment in which (12) was made lazy did not yield consistent
   improvement. *Addresses R2-C3.*
7. **Acceleration Framework / GRU** — added a two-sentence diagnosis of why
   the GRU initializer is unstable on small cases. *Addresses R1-C7.*
8. **Computational Experiments** — inserted Table 2' (speedup with bootstrap
   95% CI for the key modes), added inline CI values for the two head-line
   claims, clarified that the 14 infeasible case57 dates are excluded from
   speedup statistics. *Addresses R1-C4, R1-C9.*
9. **New subsection 4.1: Large-case follow-up on case1354pegase** — adds
   Table 6 and interpretation. *Addresses R1-C8.*
10. **New subsection 4.2: Limitations and threats to validity** — explicitly
    lists scale, screening reliability, uncertainty, split stability, and
    Gurobi auto-tuning as limitations. *Addresses R1-C10, R2-C4 and is a
    general robustness improvement.*
11. **Reproducibility paragraph** — announces the public code and data
    release. *Addresses R1-C11.*
12. **Editorial fixes** — duplicate DOI, duplicate sentence, double period,
    missing quotation marks in the bibliography, and ``These issues'' →
    ``These observations''. *Addresses R1 typ-1…typ-7.*

## What is NOT done in this revision

The following items are flagged honestly as limitations and as first-priority
future work (Sec. ``Limitations and threats to validity''):

* full `grbtune` baseline — requires its own protocol (R2-C4);
* k-fold cross-validation of the speedup (R1-C10);
* stochastic / uncertainty extension of the formulation.
